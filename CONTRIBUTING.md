# Contributing

Contributions that keep this guide accurate and useful are welcome.

## What's Needed Most

1. **Framework version updates** — Eval frameworks release frequently. If a profile's `Version Tracked` section is outdated, a PR updating it (and any changed capabilities) is high value.

2. **EU AI Act corrections** — This is a fast-moving area. If an Article mapping is wrong or incomplete, please correct it with a source citation.

3. **New use-case examples** — If your team used one of these frameworks in a scenario not covered by the decision guides, a concrete example (anonymized if needed) strengthens the guide.

4. **New framework additions** — If a framework reaches significant adoption (5k+ stars, active maintenance) and isn't covered, open an issue before writing a profile. The template is in `frameworks/README.md`.

## What Not to Submit

- Marketing copy or vendor comparisons written by employees of the frameworks covered
- Profiles without a `Version Tracked` section
- Changes to the EU AI Act module without a source citation

## PR Process

1. Fork the repo
2. Make your change on a branch named `fix/X`, `update/X`, or `add/X`
3. Verify any framework version numbers against the current release
4. Open a PR — the description should state what changed and why

## Keeping It Vendor-Neutral

This guide has no affiliation with any framework. If you work for one of the covered vendors, you're welcome to contribute accuracy corrections, but please disclose your affiliation in the PR.
